🔑 KEY CONCEPTS IN DAY 2

NextAuth.js:
  • Session management library for Next.js
  • Handles OAuth flows
  • JWT-based sessions
  • Easy Google integration

Google OAuth 2.0:
  • User authenticates with Google
  • App gets access token
  • Never see user's Google password
  • User data synced to session

SessionProvider:
  • React context wrapper
  • Provides session to all components
  • useSession() hook available everywhere
  • Handles session refresh

Protected Routes:
  • Check if user is authenticated
  • Redirect to login if not
  • useAuthProtected() hook
  • Prevents unauthorized access

❌ COMMON MISTAKES TO AVOID

1. Forgetting to wrap app with SessionProvider
   → OAuth won't work

2. Wrong Google OAuth redirect URI
   → OAuth fails with "redirect_uri mismatch"

3. .env.local not reloaded after changes
   → Credentials appear undefined

4. Using 'use client' in wrong places
   → Context won't work

5. Forgetting to update app/layout.tsx
   → Session not available to components

6. Test without restarting dev server
   → Changes to .env.local not picked up
